# Likelihood Scorer

## 这个应用是什么

Likelihood Scorer 回答一个问题：**给定一段对话上下文，模型认为某段特定回复有多"自然"？**

与 Token Explorer 不同，Scorer 不做生成——它不预测下一个词，而是对已经存在的一段文字进行"评分"。你提供完整的 user 消息和 assistant 回复，Scorer 计算模型从头到尾逐词生成这段回复的概率。

## 为什么做这个

Token Explorer 让你看到模型在每一步的选择空间，但它是"往前走"的——你选了一个词，就回不去了。

Scorer 提供了一个互补的视角：拿一段已经写好的文字，"倒过来"问模型——如果让你来写这段回复，你会给每个词多高的概率？哪些词出乎你的意料？你本来想说什么？

这个能力在以下场景中有用：

- **比较多个候选回复**：同一个问题，两段不同的回复，哪个在模型看来更自然？直接比较分数即可
- **定位"意外"的表述**：热力图中红色的 token 意味着模型觉得这个词不太对劲——也许用词不当，也许语义突然转向
- **理解模型的"偏好"**：点击任何一个 token，可以看到模型在那个位置最希望出现的前几个替代词，直观感受模型的"思路"
- **教学演示**：向他人展示"LLM 并不是一次性输出文本，而是逐词评估每个位置"

## 这不是什么

### 不是 AI 检测器

Confidence 分数高不代表这段文字是 AI 生成的。人类写的自然语言在模型看来也可能很"自然"——因为语言模型本来就是从人类文本中学来的。同样，Confidence 低也不代表不是 AI 写的——换一个模型、换一个 temperature、甚至换一种采样策略，同样的 AI 都可能产生在当前模型看来"不自然"的文本。

### 不是质量评价工具

模型给"好的。"这种敷衍回复的概率可能很高（因为它常见），但这并不意味着它是一个"好"回复。高概率 = 高预期性 ≠ 高质量。

## 核心交互流程

1. 填写 User Message（必填）——模拟用户说了什么
2. 填写 Assistant Reply（必填）——这是你想评分的那段回复
3. 可选填写 System Prompt——设定模型的角色或指令
4. 点击"Score Likelihood"
5. 后端构建完整的对话模板，跑一次前向传播，逐词计算概率
6. 前端展示：Confidence 环形图 + 技术指标 + token 热力图

## 计算过程

### 后端做了什么

1. 按照模型的对话模板（ChatML 格式），把 system prompt、user message、assistant reply 拼接成完整的 token 序列
2. 确定"reply 开始的位置"——也就是 assistant reply 的第一个 token 在整个序列中的索引
3. 将整个 token 序列喂给模型做前向传播
4. 对于 reply 部分的每个 token，从模型在上一个位置的输出中提取：
   - 模型给这个 token 的概率（它有多"预期"这个词）
   - 这个 token 在所有候选词中的排名
   - 模型最看好的前 K 个替代词及其概率
5. 汇总计算聚合指标

### 为什么只评估 reply 部分

因为我们想回答的问题是"模型在这个对话上下文下，认为这段回复自然吗"。System prompt 和 user message 是上下文——它们定义了场景，而不是被评估的对象。所以只有 assistant reply 部分的 token 参与评分计算。

## 指标详解

### Confidence 环形图（0~100%）

这是最核心的展示，计算方式为：对 reply 中每个 token 的概率取几何平均值，再乘以 100。

**它的含义**：平均来看，模型对回复中的每个词给出了多高的概率。

- 100% 意味着模型对每个 token 都 100% 确定（实际上几乎不可能）
- 60% 意味着平均每个 token 有 60% 的概率被模型选中
- 5% 意味着模型觉得大多数词都不太对

**它不是什么**：它不是模型生成这整段回复的概率。一段 50 个 token 的回复，即使每个 token 平均 60% 概率，整段被完整生成的概率也极低（0.6 的 50 次方）。Confidence 有意忽略了长度的影响，这样你才能公平地比较长短不同的回复。

### 解释标签

根据 Confidence 值给出人类可读的解释：

- **Highly Natural（≥ 60%）**：模型非常有可能生成这段回复。模型对大部分词的选择都很确定
- **Natural（≥ 35%）**：模型觉得这段回复比较符合预期
- **Plausible（≥ 15%）**：合理但不是模型的首选。某些用词或表达方式让模型有些犹豫
- **Unusual（≥ 5%）**：模型觉得不太寻常。可能有非常规的用词或出人意料的语义转折
- **Very Unlikely（< 5%）**：模型觉得非常意外。几乎每个词都不在模型的预期之内

**注意**：这些阈值是经验性的，会随模型能力波动。小模型（如 4B）整体 Confidence 偏低，因为它的词汇预测能力不如大模型精确。同样的文本在 70B 模型上可能得到更高的分数。这些标签描述的是"在当前模型看来的自然程度"，不是绝对评价。

### Perplexity（困惑度）

标准的 NLP 评估指标，计算方式为 exp(-avg_log_prob)。

直觉理解：困惑度代表"模型在每个位置平均需要从多少个等概率的候选中选择"。

- Perplexity = 1：模型完全确定，每个 token 都只有一个选择
- Perplexity = 5：相当于模型每次要从 5 个等概率的候选中挑一个
- Perplexity = 100：模型很迷茫，相当于在 100 个候选中随机选

越低越好。自然文本在好的模型上通常困惑度在 2~20 之间。

### Avg Log-Prob（平均对数概率）

每个 token 的 log 概率的平均值。负数，越接近 0 越好。这是 Confidence 和 Perplexity 的计算基础，提供最原始的数值。

### Total Log-Prob（总对数概率）

所有 reply token 的 log 概率之和。这个值会随回复长度增长而变小（更负），所以不适合直接比较长短不同的回复，但适合在定长场景下使用。

### Reply Tokens / Prompt Tokens

分别是 assistant reply 和 prompt（system + user）的 token 数量。帮助理解上下文的规模。

### Inference（推理时间）

完成整次前向传播所用的时间（毫秒）。

## Token 热力图

评分结果下方展示整段回复的逐词热力图。

### 颜色含义

- **绿色**：模型认为这个 token 在此位置高度预期（高概率）
- **黄色**：中等概率，模型觉得可以接受但不是首选
- **红色**：模型觉得很意外（低概率）

### 点击查看详情

点击任意 token 会展开详情面板，显示：

- 这个 token 的实际概率和排名
- 模型在这个位置最看好的前 K 个替代词（K 由"Alternatives per Token"参数控制）
- 如果这个 token 恰好是模型的第一选择，会标注"This was exactly what the model expected"
- 如果不是第一选择，会告诉你"模型其实更想说 X，这个词是它的第 N 选择"

这是 Scorer 最有信息量的部分——不仅告诉你"模型觉得这个词意不意外"，还告诉你"模型到底想说什么"。

## 可调参数

### System Prompt（系统提示词）

可选。设定模型的角色或指令上下文。同一段回复在不同的系统提示词下可能得到完全不同的分数——因为上下文变了，模型的"预期"也随之改变。

### Alternatives per Token（每个 token 的替代词数量）

控制热力图详情中展示多少个模型最看好的替代词。默认为 5。调大可以看到更多模型认为合适的替代表达，调小让界面更简洁。

这个参数不影响评分结果本身（Confidence、Perplexity 等），只影响详情面板展示的替代词数量。

### 为什么没有 Temperature

Token Explorer 有 temperature 参数，Scorer 刻意没有。

原因：Temperature 是一个生成时的采样策略，作用是"从概率分布里挑 token 的时候有多随机"。但 Scorer 不做生成、不做采样——它直接读取模型的原始概率来评分。如果在评分时也加入 temperature，就相当于人为扭曲了模型的真实判断，得到的分数就不再反映"模型真正怎么想"了。

Scorer 测量的是模型的**原始打分**，不应该被任何采样策略干扰。

## 请求中断机制

当你修改输入后重新点击评分，如果上一次评分还在进行中，后端会自动放弃上一次的计算，立即开始处理新的请求。这确保你看到的结果永远对应最新的输入，不会出现旧请求的结果覆盖新输入的情况。
